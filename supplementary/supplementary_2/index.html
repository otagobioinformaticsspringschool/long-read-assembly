    the REANNZ HPC File system, Working directory and Symlinks¶

    the REANNZ HPC Filesystem (For Researchers)

    All HPC platforms have custom File Systems for the purpose of general use and admin. the REANNZ HPC Filesystem looks like above

    This may look a bit obscure but thing of them as different labels for some familiar names such as Desktop, Downloads, Documents

    • /home/username is for user-specific files such as configuration files, environment setup, source code, etc. This will be the default landing file system during a login
    • /nesi/project/projectcode is for persistent project-related data, project-related software, etc
    • /nesi/nobackup/projectcode is a 'scratch space', for data you don't need to keep long term. Old data is periodically deleted from nobackup

    projectcode for this event is nesi02659. If you are to open a REANNZ HPC project for your own research, it will have a unique project code with a prefix to represent your affiliated institute and a five digit number (randomly generated).

    Therefore, full path to persistent and nobackup/scratch file systems will be in the format of

    • /nesi/project/nesi02659
    • /nesi/nobackup/nesi02659

    Symlinks (shortcuts ?)¶

    All of the Long-read assembly material will be hosted on either /nesi/nobackup/nesi02659 or /nesi/project/nesi02659 file systems. Also, each one of the attendee has an individual working space in /nesi/project/nesi02659/obss_2025/users/. Although this is great in everyway, having to type the full path to access this particular path (or having to remember it) from the default login site (/home filesystem) can be a tedious task. Therefore, we recommend creating a Symbolic link to your individual working directory from /home Think of it as a shortcut from your Desktop ðŸ™‚

    This is already done for you ðŸ˜Š - Creating a symlink from /home/$USER to project(persistent) -

    • Log into the REANNZ HPC Jupyter service as per S.1.1 : REANNZ HPC Jupyter login in REANNZ HPC Setup Supplementary material and open a terminal session
    • Let's call the symlink (shortcut) obss_2025

    • Following command will create the obss_2025 symlink from your /home directory to individual working directory in /nesi/project/nesi02659/obss_2025/users/ 

      ln -s /nesi/project/nesi02659/obss_2025/users/$USER ~/obss_2025

    • Now, you can access your working directory with 

      cd ~/obss_2025

    • Run pwd to check the current working directory
    • pwd commands on symlinks will print the "relative path" (location from where we are, rather than from the root of the file system) with respect to "absolute path" (entire path from the root directory)
    • Run the command realpath /path/you/want/to/know to show the absolute path of the symlink. i..e 
      realpath ~/obss_2025
      OR if you are already at the symlinked path 
      cd ~/obss_2025

      realpath .
